# ISO/IEC 27001 — Information Security Management System

**Canonical source:** ISO/IEC 27001:2022 (`spec:iso-iec-27001`).
**Role:** the management-system backbone under logging/audit and resilience.

## What it defines
Requirements for establishing, operating and continually improving an
Information Security Management System (ISMS): risk assessment and treatment,
Statement of Applicability, and Annex A controls (organizational, people,
physical, technological). Certification is against the clauses 4–10 management
system; Annex A is the control reference.

## Role in the crosswalk
It is the umbrella standard a security-operations program reports against: log
management, backup/DR, access control and incident response all trace to Annex A
controls and the ISMS risk treatment. For a governed AI system it is the
recognized frame a CISO and an auditor already share.

## Caveat
Certification attests the *management system* exists and is followed — not that
any specific pipeline is correctly engineered. Annex A is a reference set to
select from via the Statement of Applicability, not a mandate to implement all.
